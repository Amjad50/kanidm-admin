# Design HTML

Drop the HTML files generated by claude.ai/design into this folder, untouched.

These are **visual references** — they show what each screen should look like with the chosen design system. We don't ship them; we use them to extract structure + Tailwind classes when authoring the matching Askama template in [`../templates/`](../templates/).

Suggested filenames matching the original design briefs:

```
design/
├── 10-dashboard.html
├── 20-people-list.html
├── 22-people-detail.html
├── 60-oauth2-list.html
├── 62-oauth2-detail.html
└── ...
```

You can rename as you like — the names here are just for cross-referencing with the design briefs at `../../kanidm/design-briefs/screens/`.
